# 🚀 보스 레이드(BOSS RAID) 마스터 가이드

안녕 사용자님! 단순히 게임을 만드는 것을 넘어, **"안정적으로 빌드하고 배포하는 전문적인 방법"**을 이 문서에 모두 담았습니다. 비유와 기술적인 내용을 섞어 언제든 참고할 수 있게 정리해 드릴게요.

---

## 👨‍🍳 1. 요리에 비유한 코딩 과정

우리가 게임을 만드는 과정은 **음식을 만들어서 손님에게 배달**하는 과정과 똑같아요!

1.  **🍳 요리 중 (개발 모드)**: `npm run dev`
    *   소스 맛을 보며 재료를 계속 넣는 단계 (코드 수정 시 즉시 반영).
2.  **📦 포장하기 (빌드)**: `npm run build`
    *   배달을 위해 예쁜 상자(`dist` 폴더)에 요리를 옮겨 담는 과정.
3.  **👅 맛보기 (로컬 배포)**: `npm run preview`
    *   손님에게 보내기 직전, 포장된 요리 상태를 내 주방에서 마지막으로 확인.
4.  **🛵 배달 출발! (최종 배포)**: `npm run deploy`
    *   인터넷 세상(`Firebase Hosting`)으로 요리를 출발시킴.
    *   공식 주소: [https://project-2942784991429937034.web.app](https://project-2942784991429937034.web.app)

## 🛠 2. 상세 로컬 빌드 방법 (Step-by-Step)

내 컴퓨터에서 코드를 '배포용 파일'로 만드는 정확한 순서입니다.

1.  **터미널 열기**: VS Code 하단의 `Terminal` 탭을 확인합니다.
2.  **동작 중인 서버 종료**: 만약 `npm run dev`가 돌아가고 있다면 `Ctrl + C`를 눌러 멈춥니다.
3.  **빌드 명령어 입력**:
    ```bash
    npm run build
    ```
4.  **결과물 확인**: 내 프로젝트 폴더 안에 `dist`라는 폴더가 새로 생겼는지 확인합니다. 이 폴더 안의 파일들이 실제 웹사이트가 됩니다.
5.  **로컬 서버 실행**:
    ```bash
    npm run preview
    ```
    *   터미널에 뜨는 `http://localhost:4173` 주소를 클릭해 화면이 잘 나오는지 봅니다.

## ✅ 3. 배포 전 체크리스트 (현황 파악용)

배포 버튼을 누르기 전, 다음 사항들을 체크하면 실수를 99% 줄일 수 있습니다!

- [ ] **에러 확인**: 터미널 창에 빨간색 `Error` 메시지가 떠 있지 않은가?
- [ ] **로컬 테스트**: `npm run preview`로 실행했을 때 게임 오버나 체력바가 정상 작동하는가?
- [ ] **경로 확인**: `vite.config.js`에 `base: '/book/'` 코드가 정확하게 들어있는가? (이게 없으면 이미지가 깨집니다.)
- [ ] **저장 여부**: 모든 파일 상단에 흰색 동그라미(저장 안 됨 표시)가 없는가? (`Ctrl + S` 완료 상태)
- [ ] **서버 중복**: 다른 터미널 창에서 같은 포트(예: 3000, 5173)가 이미 돌아가고 있지는 않은가?

## ⚠️ 4. 실패 사례 및 예외 케이스 정리

문제가 생겼을 때 당황하지 말고 아래 해결책을 보세요.

| 문제 현상 | 원인 | 해결 방법 |
| :--- | :--- | :--- |
| **흰 화면만 나와요** | 파일 경로 문제 (`base` 설정 누락) | `vite.config.js`의 `base` 경로를 다시 확인하세요. |
| **사이트를 찾을 수 없습니다** | 서버에 파일이 업로드되지 않음 | `npm run deploy` 명령어를 실행하여 파일을 전송하세요. |
| **PowerShell 보안 오류** | 윈도우 스크립트 실행 방지 정책 | `npx.cmd`를 사용하거나 보안 정책을 해제(`Set-ExecutionPolicy`)하세요. |
| **Failed to authenticate** | 파이어베이스 로그인 세션 만료 | `npx firebase login`을 입력하여 다시 로그인하세요. |
| **배포 후 보스가 안 보여요** | 코드 누락 또는 빌드 오류 | 코드를 다시 저장하고 `npm run build` 후 배송하세요. |
| **배포했는데 예전 버전만 나옴** | 브라우저 캐시 문제 | `Ctrl + Shift + R`을 눌러 강력 새로고침 하세요. |
| **명령어가 안 먹혀요** | 오타 또는 폴더 위치 틀림 | `book2` 폴더 안인지 확인하고 `npm.cmd`를 사용하세요. |
| **Port already in use** | 이전 서버가 아직 켜져 있음 | 터미널에서 `Ctrl + C`로 서버를 끄고 다시 시도하세요. |

## ☁️ 5. 클라우드 데이터 동기화 (Firestore)

이제 우리 게임의 데이터는 내 컴퓨터가 아닌 **구글 클라우드**에 저장됩니다.

*   **저장 원리**: 보스를 물리치면 기록이 즉시 파이어베이스 데이터베이스(Firestore)로 전송됩니다.
*   **장점**: 
    *   집에서 하던 게임 기록을 밖에서도 똑같이 확인할 수 있습니다.
    *   전 세계 사람들의 '명예의 전당' 기록을 한곳에 모을 수 있습니다.
*   **관리**: 데이터 확인은 [Firebase Console](https://console.firebase.google.com/)의 **Firestore Database** 메뉴에서 직접 하실 수 있습니다.
